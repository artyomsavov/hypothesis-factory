# 🏭 Hypothesis Factory 

Адаптивный ИИ-конвейер для генерации, оценки и ранжирования научно-исследовательских гипотез в материаловедении. Разработано в рамках хакатона "Фабрика гипотез".

## 🎯 Суть подхода

Мы отказались от хрупких end-to-end решений в пользу жесткой модульной архитектуры `Generator-Critic` с гибридным RAG. Главный фокус системы — **максимальная отказоустойчивость**. 

![Архитектура конвейера](docs/architecture.gif) 

## Требования и установка

Проект использует [uv](https://github.com/astral-sh/uv) для быстрого управления зависимостями.

1. Клонируйте репозиторий:
```bash
git clone [https://github.com/artyomsavov/hypothesis-factory.git](https://github.com/artyomsavov/hypothesis-factory.git)
cd hypothesis-factory
```

2. Установите зависимости и создайте виртуальное окружение:
```bash
uv sync
```

## 🛠 Как это работает под капотом

1. **Ingestion:** Сырые файлы парсятся и очищаются. Битые файлы изолируются.
2. **Retrieval (RAG):** Текст нарезается на чанки. Извлекается релевантный контекст под задачу (снижение себестоимости, повышение жаропрочности и т.д.).
3. **Generator:** Первая LLM извлекает факты и генерирует набор черновых гипотез со строгой привязкой к источникам (цитированием).
4. **Critic:** Вторая LLM выступает в роли критика. Она отсеивает физический бред, скорит гипотезы по метрикам реализуемости и формирует итоговый рейтинг.
